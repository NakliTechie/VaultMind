🧠 VaultMind — Help

An interactive force-directed graph of your entire vault. Each node is a note; each edge is a [[wikilink]].

  • Click a node — opens the note preview panel.
  • Drag a node — pin it; drag away to move it freely.
  • Scroll / pinch — smooth zoom centred on cursor.
  • Drag canvas (2D) — pan the view.
  • 3D button — switch to 3D perspective. Drag to rotate on all axes; the blob stays centred.
  • ⏸ Pause — freeze the simulation.
  • ⊞ Fit — reset zoom to fit all nodes.
  • + / − — add / remove a random connection (demo).

Node colours reflect clusters detected by the force layout. Red = orphan (no links). Pink = MOC.

Ask questions about your vault. Context is pulled from the selected note and its neighbours (micro-RAG).

  • Local (WebGPU) — runs a small LLM entirely in your browser. No data leaves the device. Requires a WebGPU-capable browser (Chrome 113+).
  • API — connects to any OpenAI-compatible endpoint. Works with OpenRouter, Groq, LM Studio, Ollama, and more.
  • Click a node then ask — chat automatically scopes context to that note and its 1-hop neighbours.
  • 💬 button on the note panel — focuses the chat on the current note.
  • 🌐 button in the chat bar — toggle a live web search to augment answers.
  • Settings collapse automatically when a model is loaded.

LM Studio / Ollama? Enable CORS: in LM Studio open Local Server → Server Settings → Enable CORS. For Ollama restart with OLLAMA_ORIGINS="*" ollama serve.

Click any note to open the preview panel, then press ✏️ to edit it in place.

  • Toolbar — B / I wrap selection; H1–H3 prefix the current line; `c` inline code; ``` code block; 🔗 link; • / 1. / ☐ list items; — rule; " quote.
  • Ctrl / ⌘ + B / I / S — bold, italic, save shortcuts.
  • Autosave — tick the checkbox; saves to disk 1.5 s after each keystroke.
  • Save — writes back to the original .md file (requires the vault to be opened as a folder, not a ZIP).
  • Discard — restores the last saved version.
  • Unsaved draft — if you switch notes without saving, your edits are kept in memory (amber ● unsaved draft badge). Come back and continue, or Discard.
  • A small amber dot appears next to the note name in the sidebar while unsaved changes exist.

The search box in the Notes sidebar searches full note bodies, not just titles.

  • Multi-word query — each word is scored independently. A note mentioning all terms more often ranks higher.
  • Title hits are weighted 10× body hits, so a note whose name matches always floats to the top.
  • Snippets — a short extract around the first match is shown below each result, with your query terms highlighted.
  • No query — list reverts to link-degree order (most connected notes first).
  • Tip: search is instant and entirely in-memory — no network, no indexing step.
